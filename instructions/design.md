# 要件

- 0→1開発を前提とする
- Coding Agentがそのまま実装できる粒度まで具体化する
- Context Window節約を考慮し、適切にドキュメントを分割する
- README.mdには概要のみを記載し、詳細は別ファイルへ分離する
- AGENTS.mdを作成し、Agent向けの作業ルールを記載する
- MVPと将来機能を分離する
- 不明点は「未確定事項」として明記する
- 推測した内容は「仮定」として明記する
- テストは必須とする

# 含める内容

サービスの内容や規模に応じて不要なものは出力しなくても良い

- サービス概要
- 目的・ゴール
- システム構成
- インフラ構成
- 機能一覧
- 実装TODOリスト
- データモデル
- API設計
- テスト戦略
- 開発フロー

# 出力形式

以下のファイル構成をベースに複数のファイルをraw markdownとして出力しなさい

```text
README.md

docs/
├── 01_overview.md
├── 02_requirements.md
├── 03_architecture.md
├── 04_features_todo.md
├── 05_database.md
├── 06_api.md
├── 07_frontend.md
├── 08_backend.md
└── 09_infrastructure.md
```

## README.md

以下のみ記載してください。

- ドキュメント一覧
- 推奨読書順

- テスト必須ルール
- 開発開始手順

## docs/04_features_todo.md

最重要ファイルです。機能一覧を実装タスクへ分解してください。

以下の形式を使用してください。

```md
# MVP

## 認証

- [ ] ユーザー登録API
- [ ] ログインAPI
- [ ] ログアウトAPI

受け入れ条件

- 登録できる
- ログインできる
- セッションが維持される
```

各機能に

- 優先度
- 依存関係
- 受け入れ条件

を記載してください。
